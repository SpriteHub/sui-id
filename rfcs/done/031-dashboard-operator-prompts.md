# RFC 031 — Dashboard operator prompts and active session count

**Status.** Proposed  
**Priority.** Medium-High. Operators cannot see at a glance whether SMTP,
HIBP, or `cookie_secure` are properly configured. This is the "next operator
action" visibility that the dashboard information policy (RFC 017 § 5) mandates.  
**Source.** UI/UX design document P.8; RFC 017 § 5.  
**Touches.** `crates/sui-id-core/src/dashboard.rs`,
`crates/sui-id-web/src/pages.rs` (`DashboardData`, `render_dashboard`),
`crates/sui-id/src/handlers/admin.rs`.

## Problem

`DashboardData` currently carries: `admin_username`, `user_count`,
`client_count`, `issuer`, `sparkline`. Missing:

1. **Active session count** — design explicitly requires it alongside user/client counts.
2. **SMTP configured flag** — unset = forgot-password is silently broken.
3. **HIBP mode** — `Off` warrants a visible nudge.
4. **`cookie_secure` flag** — `false` in production is a security defect.
5. **Operator action prompts** — shown only when the condition is true; absent when resolved.
6. **JWKS / Discovery reachability** — "200 OK from the public endpoints".

## Design: operator action prompts

Prompts appear as a `flash warn` banner (or a compact checklist section)
above the stats cards. Each prompt disappears when its condition resolves:

| Condition | Prompt text |
|---|---|
| SMTP not configured | "Forgot-password email is disabled. Configure SMTP in Settings → Email." |
| HIBP mode is `Off` | "Password breach checking is off. Enable in Settings → Authentication." |
| `cookie_secure = false` | "Cookie Secure flag is off. Set to `true` in production (Settings → Security)." |
| Master key generated by sui-id | "Back up `sui-id.key` — if lost, encrypted data cannot be recovered." |

A clean deployment (all conditions false) shows zero prompts.

## Changes

### `DashboardData`

```rust
pub struct DashboardData {
    pub admin_username:       String,
    pub user_count:           usize,
    pub client_count:         usize,
    pub active_session_count: usize,    // NEW
    pub issuer:               String,
    pub sparkline:            DashboardSparkline,
    // Operator prompts (each None = condition not triggered)
    pub warn_smtp_not_configured:  bool,  // NEW
    pub warn_hibp_off:             bool,  // NEW
    pub warn_cookie_insecure:      bool,  // NEW
    pub warn_backup_master_key:    bool,  // NEW (key was auto-generated)
    pub lang:                 Locale,     // NEW (for RFC 029 i18n)
}
```

### Handler changes

`handlers/admin.rs::dashboard` queries:
- `sessions::count_active_total(db)` → `active_session_count`
- `server_settings::get(db).hibp_mode` → `warn_hibp_off`
- `smtp_config::get(db).is_none()` → `warn_smtp_not_configured`
- `app.config.server.cookie_secure` → `warn_cookie_insecure`
- master key origin flag (stored in `server_settings`) → `warn_backup_master_key`

### UI changes

- Add a third stat card: **Active sessions** (alongside users and clients).
- Add an operator prompt section above the cards. Uses `.flash.warn` styling.
  Shows each warning as a paragraph with a direct link to the relevant settings tab.
- Dashboard title and labels converted to `t.` references (RFC 029 alignment).

dashboard_warn_smtp / dashboard_warn_hibp / dashboard_warn_cookie_insecure

## Tests

- Unit: `count_active_total` returns correct count across users.
- E2E: Dashboard shows SMTP prompt when smtp_config is absent.
- E2E: Dashboard shows no prompts when SMTP configured + HIBP warn + cookie_secure=true.
